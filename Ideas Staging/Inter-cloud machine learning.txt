Inter cloud machine learning

人在作判断的时候和机器学习很像，选取feature然后通过一些运算。feature选得好不好是判断准确性的关键。比如面试官看考生，是衣着呢、谈吐呢、答题成绩呢等等。另外比如对新闻的判断，对某个商品的判断，对某个事、某个人的判断。feature判断的例子，渗透生活的方方面面。

机器学习对feature的选择非常小心，feature应该有一定的数量和良好的质量。但对人而言，我要吐槽的是，不少人居然看到一个feature就敢下判断，而且大多数人也不自觉地经常这样做。和机器学习比，这是多么轻率的判断方式，但事实是人们照样活到了今天，活得好好的。

除了人有大量的经验（活了几十年），人的观念随着更多的经验逐渐更新，我这里要说的、非常重要的是，人会交流，社群之间在不停交流。人们每天都会互相交流，谈些重要不重要、有关或没关的事情，即闲聊。生活大爆炸里的Sheldon认为闲聊是没用的事情。但我的观点是，闲聊对社群及其重要，是人们之间不断的sync up。对于其作用，只举一例。人的语言赖以词汇，但词汇的含义模糊而难以给出明确定义，那为什么人与人之间可以交流畅通无阻，并且“确定”能够通过词汇理解对方的意思？词汇的意义，为什么不能在人之间漂变至各不相同无法交流？因为社群总在sync up。不断的sync up使得模糊的词汇含义在社群中维持了模糊的一致性。

——并且，为什么轻率判断的人们获得好好的？因为他们活在人群中，因为他们总在sync up。把人看作机器学习，每个人各自有各自的模型。交流，或者说sync up这种行为，使得社群里机器学习的质量都上升了，单独的模型不再容易完蛋，社群形成强大的智力。所以，我要说的是，交流、sync up是改进机器学习的一条充满希望的路。

现在，大家各自搞机器学习，各自训练模型。世界上，机器学习个体的数量已经相当多，但我们没有它们间的交流或sync up。我认为全世界机器学习之间的inter-connect、交流和sync up是大幅改进它们的希望之路。以往的实践有侧面上对这个想法的支持：把多个模型联合起来使用，往往比单个模型有更好的效果，见A few useful things to know about machine learning - http://homes.cs.washington.edu/~pedrod/papers/cacm12.pdf - section ”10. LEARN MANY MODELS, NOT JUST ONE“。

我想建立机器学习的世界社群、机器学习的inter-connect，。cisco提出了inter-cloud的想法，连接世界上的所有云，就像internet当年连接各企业的私有网一样。机器学习早晚搬到云上进行，那么机器学习的inter-connect可以作为inter-cloud的一种解释。另一方面，机器学习如今的使用越来越普遍，我认为它早晚渗透编程/软件的方方面面，就如同人类交替使用确定性思维和模糊性思维。"machine learning inter cloud"可能成为人类的基础设施，如互联网一般。为了让大家加入"machine learning inter cloud"，我想可以仿照开源软件的方式开始。为了让商业公司也加入，可以仿照openstack的成功方式。我觉得"machine learning inter cloud"有潜力制造出惊人的智能，也许有一天我们能把天网造出来。如果这个Idea挂了，我还可以拿它写科幻小说。

对于机器学习之间的交流，说不定可以利用我另一个Idea——“模仿词汇的交流方式”。

人类交流时，通过词汇互相传达意思。但词汇只是个图形符号，我的看法是，”内容“本身根本不在词汇里，传输过程中，”内容“没有被传输。交流的双方人，本身已经拥有大量的publich knowledge，从词汇含义到生活经验，几十年的积累。词汇传递的不是”内容“，而是像一把钥匙到达对方，然后激活public knowledge的网络，”内容“就传递到了。

相反地，目前计算机网络，我们的思路就是把”内容“传过去。新的想法就是，计算机都有prebuild的public knowlege，我们不再传输”内容“了，用”词汇“或钥匙传过去，对方就可以得到”内容“。总之，仿照人类交流的方式吧。计算机的public knowledge可以是几十年培养出来的经验系统，出厂预装。经验什么的，可以和机器学习沾边了。而”词汇“，也许是一种机器学习把现象进行聚类后的产物。

这个想法，与”machine learning inter cloud“正好沾上关系，要求通信，而且基础在机器学习上。让我们造出天网吧～
